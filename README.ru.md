# claw0

**От нуля до единицы: Построение шлюза AI Agent**

> 10 прогрессивных разделов -- каждый раздел это отдельный, готовый к запуску Python файл.
> 4 языка (English, 中文, 日本語, Русский) -- код + документация рядом.

---

## Что это?

Большинство туториалов по агентам останавливаются на "вызвать API один раз". Этот репозиторий начинает отсюда, с того самого while цикла, и доводит вас до production-ready шлюза.

Постройте минимальный шлюз AI Agent с нуля, раздел за разделом. 10 разделов, 10 основных концепций, примерно 7000 строк Python. Каждый раздел вводит ровно одну новую идею, сохраняя весь предыдущий код. После всех 10 разделов вы сможете уверенно читать production codebase OpenClaw.

```sh
s01: Agent Loop           -- Основа: while + stop_reason
s02: Tool Use             -- Дать модели возможность вызывать инструменты: dispatch table
s03: Sessions & Context   -- Персистентность сессий, обработка переполнения контекста
s04: Channels             -- Telegram + Feishu: полные конвейеры каналов
s05: Gateway & Routing    -- 5-уровневое связывание, изоляция сессий
s06: Intelligence         -- Душа, память, навыки, сборка промптов
s07: Heartbeat & Cron     -- Проактивный agent + плановые задачи
s08: Delivery             -- Надежная очередь сообщений с backoff
s09: Resilience           -- 3-слойная луковица повторов + ротация аутентификации
s10: Concurrency          -- Именованные полосы сериализуют хаос
```


## Зависимости разделов

```
s01 --> s02 --> s03 --> s04 --> s05
                 |               |
                 v               v
                s06 ----------> s07 --> s08
                 |               |
                 v               v
                s09 ----------> s10
```

- s01-s02: Основа (нет зависимостей)
- s03: Строится на s02 (добавляет персистентность к tool loop)
- s04: Строится на s03 (каналы производят InboundMessage для сессий)
- s05: Строится на s04 (маршрутизирует сообщения каналов к агентам)
- s06: Строится на s03 (использует сессии для контекста, добавляет слои промпта)
- s07: Строится на s06 (heartbeat использует soul/memory для промпта)
- s08: Строится на s07 (выход heartbeat идет через очередь доставки)
- s09: Строится на s03+s06 (переиспользует ContextGuard для переполнения, конфиг модели)
- s10: Строится на s07 (заменяет единственный Lock на систему именованных полос)

## Путь обучения

Каждый раздел добавляет ровно одну новую концепцию. Весь предыдущий код остается нетронутым:

```
Этап 1: ОСНОВА       Этап 2: СВЯЗНОСТЬ       Этап 3: ИНТЕЛЛЕКТ       Этап 4: АВТОНОМИЯ      Этап 5: PRODUCTION
+----------------+   +-------------------+   +-----------------+   +-----------------+   +-----------------+
| s01: Loop      |   | s03: Sessions     |   | s06: Intelligence|   | s07: Heartbeat  |   | s09: Resilience |
| s02: Tools     |-> | s04: Channels     |-> |   душа, память, |-> |   & Cron        |-> |   & Concurrency |
|                |   | s05: Gateway      |   |   навыки, про.  |   | s08: Delivery   |   | s10: Lanes      |
+----------------+   +-------------------+   +-----------------+   +-----------------+   +-----------------+
 цикл + dispatch     персист + маршрут.      личность + вспомн.     проактив + надежность  повтор + сериализ.
```

## Детали разделов

| # | Раздел | Основная концепция | Строк |
|---|--------|-------|-------|
| 01 | Agent Loop | `while True` + `stop_reason` -- вот это agent | ~175 |
| 02 | Tool Use | Инструменты = schema dict + handler map. Модель выбирает имя, вы ищете в таблице | ~445 |
| 03 | Sessions | JSONL: дописываем при записи, воспроизводим при чтении. Слишком большой? Суммируем старые части | ~890 |
| 04 | Channels | Каждая платформа отличается, но все производят один и тот же `InboundMessage` | ~780 |
| 05 | Gateway | Таблица связывания отображает (channel, peer) на agent. Побеждает наиболее специфичный | ~625 |
| 06 | Intelligence | Системный промпт = файлы на диске. Меняйте файлы, меняйте личность, не меняйте код | ~750 |
| 07 | Heartbeat & Cron | Поток таймера: "должен ли я запуститься?" + поставить работу наряду с сообщениями пользователя | ~660 |
| 08 | Delivery | Сначала пишем на диск, потом пытаемся отправить. Крахи не могут потерять сообщения | ~870 |
| 09 | Resilience | 3-слойная луковица повторов: ротация auth, компактирование переполнения, tool-use loop | ~1130 |
| 10 | Concurrency | Именованные полосы с FIFO очередями, отслеживание поколений, Future-based результаты | ~900 |
